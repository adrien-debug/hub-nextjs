## CE QUE L'ADMIN FAIT ACTUELLEMENT (fonctionnalités à conserver 1:1)

### Login
- Formulaire username/password → POST /api/auth/login
- Token stocké en localStorage
- Auto-login si token existant

### Dashboard — Onglet Projets
- 4 stat cards (Total, Publiés, Live, Pipeline)
- Filtres par statut (all/live/coming/future/archived)
- Table : nom, statut badge, catégorie, port, stack, actions (publier/masquer, supprimer)

### Dashboard — Onglet Incubation
- 4 stat cards (Total startups, Actives, Graduées, Phases)
- Filtres par statut startup (all/applicant/screening/accepted/active/graduated/alumni/rejected)
- Table : nom + fondateurs, statut, phase courante X/28, cohorte, score, bouton "Ouvrir"

### Vue Détail Startup (quand on clique "Ouvrir")
- Header avec bouton retour, nom, badge statut, phase courante
- 4 info cards (Fondateurs, Catégorie, Cohorte, Score)
- Barre de progression globale (X/28 — Y%)
- Blocs Problème / Solution
- **28 phases timeline expandable** :
  - Clic = expand : description, critères validation, livrables requis
  - Textarea notes/appréciations (sauvegarde onBlur → PATCH API)
  - Textarea blockers (sauvegarde onBlur → PATCH API)
  - Dates (début, fin, durée typique)
  - Boutons d'action : Démarrer / Valider / Bloquer / Débloquer
  - Auto-avance : valider une phase passe la suivante en in_progress

## CE QUE TU DOIS AMÉLIORER (UNIQUEMENT LE VISUEL)

L'admin actuel est fonctionnel mais **trop plat** — c'est une table avec des textes, pas assez visuel pour un dashboard admin premium.

### Direction créative souhaitée :
1. **Plus de densité visuelle** — utiliser les design tokens (accent glow, grain, borders subtiles) pour donner de la profondeur
2. **Cards au lieu de tables brutes** pour les projets et startups — layout grid responsive
3. **Pipeline visuel** pour les 28 phases — pas juste une liste expandable mais un vrai tracker visuel (barres, dots connectés, couleurs par statut)
4. **Micro-interactions** — hover states, transitions, focus rings accent
5. **Hiérarchie typographique** adaptée admin : pas de titres géants (pas de text-6xl, text-8xl etc.), rester en text-sm à text-xl max, mais utiliser weight/spacing/color pour créer la hiérarchie
6. **Stat cards plus riches** — mini sparklines en CSS, icônes, progress rings, ou compteurs animés
7. **Le login** doit rester sobre mais premium — centré, dark, accent sur le CTA

### Ce qu'il NE faut PAS faire :
- Pas de titres landing page géants (text-4xl+)
- Pas de nouveau composant externe — tout dans le même fichier page.tsx
- Pas changer la logique (fetch, auth, state management)
- Pas ajouter de dépendances (pas de charting lib, pas de UI lib)
- Pas de sidebar/drawer complexe — garder la structure tabs
